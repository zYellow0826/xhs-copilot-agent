# 选题

- 单一卖点：一篇笔记只讲一个核心利益点，不要堆功能
- 痛点钩子：从"目标人群当下最焦虑的场景"切入，而不是从商品本身切入
- 差异定位：避开同品类红海词，找细分场景下的"窄而深"切角

# 标题（硬规则）

- 严格 ≤ 20 字
- 必须包含至少一个钩子：数字 / 反差 / 痛点 / 身份标签
- 不堆叠 emoji（首字符或行末最多 1 个）
- 不出现违禁词：最/第一/绝对/100%/根治/神器

# 正文

- 200-500 字
- 开头 3 行内点出"这篇能帮你解决什么问题"
- 短段落，每段不超过 3 行
- 适度 emoji（每段 0-2 个，正负情绪点缀用）
- 收尾给一个明确的行动指令（评论 / 私信 / 收藏 / 进店）

# 标签

- 3-10 个
- 大词（流量基本盘）+ 长尾词（精准用户）混搭
- 不要全是品牌词或商品名

# 封面文案

- 一行短句，≤ 12 字
- 信息密度优先于情绪
- 与标题不重复（封面给"利益"，标题给"钩子"）
