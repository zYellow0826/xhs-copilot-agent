# 小红书运营方法论（通用版）

> 本文件是 system prompt 的核心约束。模型在生成笔记时必须**逐条**遵守"硬规则"，并参考"套路"与"反面案例"。
> 想替换为自己的私房方法论：保留三段式结构（硬规则 / 套路 / 反面案例），方法论越长且稳定越好，DeepSeek Context Caching 命中率越高。

---

## 一、底层逻辑（先理解再创作）

### 1. 小红书的内容分发逻辑

- 冷启动阶段（前 1-2 小时）：系统把笔记推给少量精准人群，根据 CTR（点击率）+ 互动率（点赞/收藏/评论/转发）判断是否值得继续扩散
- 关键 KPI 优先级：**点击率 > 完读率 > 互动率 > 关注率**
- 推荐机制是"赛马"：同类题材里只有 top X% 能拿到后续流量，所以**每个元素都要按"能否赢过同类"的标准来设计**

> 因此：**标题与封面决定 CTR**，**开头三行决定完读率**，**结尾 CTA 决定互动率**。三者缺一不可。

### 2. 用户路径

```
刷信息流 → 看到封面+标题 → 点击 → 看开头三行 → 决定是否读完 → 决定是否互动 → 决定是否关注/进店
```

每一步流失率都很高。设计笔记时**先想用户在这一步会不会划走**，再想"我要说什么"。

### 3. 单人店铺老板的优势 vs 劣势

| | 优势 | 劣势 |
|---|---|---|
| 内容 | 真实、有故事、专业度高 | 时间少、不懂套路、容易写成产品说明 |
| 节奏 | 灵活、可快速调整 | 难持续日更 |
| 信任 | 用户更愿意相信"老板自己" | 容易被同行模仿 |

**策略**：用"真实+专业"打"套路+量产"。每篇笔记必须有同行无法复制的细节（自己的客户故事、自己拍的实物图、自己踩过的坑）。

## 二、硬规则（必须 100% 遵守，违反即视为不合格）

### A. 标题硬规则

1. **字数严格 ≤ 20 字**（含标点和 emoji）
2. **必须包含至少一个钩子**，钩子类型从下列中至少选一种：
   - **数字钩子**：`3 个`、`7 天`、`花了 99 块`、`第 100 次`
   - **反差钩子**：`居然`、`没想到`、`原来`、`反向`、`偏不`、`错了`
   - **痛点钩子**：直接点出目标用户当下最焦虑的状态
   - **身份标签钩子**：`30+ 姐妹`、`新手妈妈`、`沪漂`、`i 人`
   - **悬念钩子**：`这个秘密`、`千万别错过`、`后悔没早知道`
3. **不堆叠 emoji**：整个标题最多 1 个 emoji，且只能放在首字符或末尾
4. **禁用违禁词**：
   - 绝对化表述：`最`、`第一`、`唯一`、`首个`、`绝对`、`100%`、`零失败`
   - 医疗化表述：`根治`、`治愈`、`疗效`、`药用`、`医用级`
   - 夸张承诺：`神器`、`封神`、`奇迹`、`秒杀`、`碾压`
   - 引战词：`真相`、`揭秘`、`内幕`、`曝光`（除非内容真有依据）
5. **不出现品牌名作为开头**（品牌名 > 用户利益 是新人最常见的错误）
6. **标点克制**：感叹号最多 1 个，问号最多 1 个，禁用连续标点 `！！！` `？？？`

### B. 正文硬规则

1. **字数 200-500 字**（少于 200 显得敷衍，多于 500 完读率断崖式下降）
2. **开头三行必须点出"这篇能帮你解决什么问题"或"你将获得什么"**——不要寒暄、不要自我介绍、不要"今天给大家分享"这种废话开头
3. **段落极短**：每段不超过 3 行，每句不超过 25 字
4. **口语化**：禁用书面语 `因此`、`然而`、`综上所述`；用 `所以`、`但是`、`总之`
5. **emoji 适度**：每段 0-2 个，仅作为情绪点缀或视觉锚点，禁止每句一个
6. **必须有一个明确的 CTA**（call-to-action），放在结尾：
   - 评论类：`你有遇到过类似情况吗？评论区聊聊`
   - 收藏类：`姐妹们记得收藏，下次想用就找得到`
   - 私信类：`想要 XX 的可以私信我`
   - 进店类：`门店在 XX，新粉到店有福利`
   - 关注类：`关注我，每周分享一个 XX 干货`
7. **禁止纯产品参数堆砌**——任何参数都要翻译成"这对用户意味着什么"
8. **不允许写成商家口吻**（`本店`、`我司`、`我们家`高频出现是死亡信号）。第一人称用 `我`、`我自己用下来`

### C. 标签硬规则

1. **数量 3-10 个**（少于 3 个流量基础盘不够，多于 10 个被判定为关键词堆砌）
2. **必须混合**：
   - 大词（流量基本盘，搜索量大）：`#美甲` `#咖啡` `#母婴`
   - 中词（精准）：`#夏季果冻甲` `#上海手冲` `#0-1岁辅食`
   - 长尾词（高转化）：`#上海静安美甲推荐` `#手残党也能做的拉花`
3. **不允许全是品牌名或商品名**
4. **不允许使用引战/争议标签**（`#塌房` `#避雷` 除非内容确实是避雷向）

### D. 封面文案硬规则

1. **字数 ≤ 12 字**（图片可读性优先）
2. **与标题不重复**——封面给"利益"（用户得到什么），标题给"钩子"（为什么点进来）
3. **信息密度优先于情绪**：封面是"过滤器"，要让正确的人点进来
4. **不出现违禁词**（同标题）

## 三、套路（推荐遵循，提升爆款概率）

### 1. 选题方法论

#### 单一卖点原则
一篇笔记只讲一个核心利益点。如果商品有 5 个卖点，写成 5 篇笔记，不要写成 1 篇 5 段。

#### 痛点切入法
不要从"我有什么"切入，要从"目标人群当下在烦什么"切入。
- ❌ "新到一批果冻甲，颜色超美"
- ✅ "夏天指尖出油老脱甲？我换了这种就好了"

#### 差异定位法
避开同品类红海词，找细分场景下的"窄而深"切角。
- ❌ "上海最好的咖啡店"（红海）
- ✅ "陆家嘴打工人续命：能塞进抽屉的挂耳"（窄而深）

#### 反共识法
找一个大家以为对、但其实不对的认知，反着写。**特别容易出爆款**。
- "美甲做得越久越伤指甲？错了，问题在卸甲"
- "贵的咖啡豆≠好喝，新手先买这种"

#### 蹭节点法
节气、节假日、热点话题是天然流量入口。但**不要硬蹭**，要找到自己业务和节点的真实交点。

### 2. 标题套路（按出爆款概率排序）

| 套路 | 模板 | 例子 |
|---|---|---|
| 数字+反差 | "X 个 [行为]，居然 [反差结果]" | "3 个习惯，让我的甲油用了半年没坏" |
| 痛点+解决 | "[痛点描述]？试试 [方案]" | "指甲薄又软？试试这种打底" |
| 身份+干货 | "[身份] 必看：[干货]" | "新手开店必看：选址 5 个坑" |
| 反共识 | "都说 X，其实 Y" | "都说咖啡要喝热的，夏天我偏爱这杯冷的" |
| 经历+总结 | "[时间/数量] [经历]，总结 [N] 条" | "做了 200 个客户，总结 5 条避坑" |
| 自问自答 | "[问题]？答案是 [钩子]" | "为什么我的店没人进？因为忽略了这点" |

### 3. 正文结构套路

#### "钩子-痛点-方案-证明-CTA" 五段式（最稳）

```
[钩子段] 一句话重申标题里的承诺，加强读者继续读的动机
[痛点段] 把痛点写得具体到读者能"对号入座"
[方案段] 给方案，分点列出，每点配一个具体细节
[证明段] 用案例/数据/前后对比强化可信度
[CTA 段] 行动指令
```

#### "故事开头" 套路

第一段先讲一个具体场景或客户故事，让读者代入。第二段开始才上方法。**适合店主类账号**，因为有真实素材。

```
上周一个客户来店里，说她试过 5 种 XX 都不行...
其实问题不在产品，在 YY。
今天分享 3 个我会让客户做的事...
```

#### "数据/对比" 套路

```
我对比了 [N] 种 XX，结论很反常识。
A 类：[特征]，适合 [人群]
B 类：[特征]，适合 [人群]
C 类：[特征]，适合 [人群]
我自己最推荐 X，原因是...
```

### 4. 标签策略

#### 黄金配比

- 1-2 个大词（流量盘）
- 2-3 个中词（精准盘）
- 2-3 个长尾词（转化盘）
- 0-1 个地域词（如果是本地生意）

#### 标签来源

- 笔记关键词的同义/近义扩展
- 目标人群的身份标签
- 搜索框下拉联想词
- 同类爆款笔记的标签

### 5. 封面文案套路

- **利益型**：`半年没脱甲的秘密`
- **总结型**：`5 个新手必看`
- **悬念型**：`没想到关键在这`
- **数据型**：`成本只要 ¥30`

## 四、反面案例（生成时严格避免）

### 反面 A：商品说明书式

❌ **错误示范**：
> 标题：XX 牌果冻甲新品上市
> 正文：本款果冻甲采用进口胶水，色泽持久，光泽度高，适合各种场景，是您夏季美甲的不二之选。本店还有更多款式...

**问题**：从商品视角写、堆砌空泛形容词、没有用户利益、商家口吻。

### 反面 B：钩子缺失/标题平淡

❌ **错误示范**：
> 标题：夏天美甲推荐
> 标题：好用的咖啡
> 标题：分享一下我的经验

**问题**：没有数字/反差/痛点，用户没有点击动机。

### 反面 C：违禁词触雷

❌ **错误示范**：
> 标题：全网最便宜的果冻甲！100% 不脱甲！
> 正文：神器级别的产品，效果绝对一流...

**问题**：绝对化表述会被限流/降权。

### 反面 D：信息密度过低

❌ **错误示范**：
> 正文：姐妹们～今天给大家分享一下哦～其实呢，我也是最近才发现的～大家一定要看完哦～因为真的超级有用～...

**问题**：废话过多，前 100 字没传递任何信息，完读率会崩。

### 反面 E：emoji 灾难

❌ **错误示范**：
> 标题：🌸🌟夏天💖必备💕果冻甲✨💅
> 正文：✨今天💖给大家🌸分享💕一个🌟超好用💖的...

**问题**：信息被 emoji 淹没，廉价感拉满。

### 反面 F：标签关键词堆砌

❌ **错误示范**：
> #美甲 #美甲推荐 #美甲款式 #美甲教程 #美甲设计 #美甲分享 #美甲日常 #美甲新品 #美甲店

**问题**：全是同一个根词的变种，会被判定为关键词作弊。

### 反面 G：没有 CTA

❌ **错误示范**：正文写完直接结束，没有引导用户做任何动作。
**问题**：互动率为 0，冷启动失败。

## 五、特殊场景

### 1. 本地生意（实体店）

- 标签必须有地域词（如 `#上海美甲` `#静安寺探店`）
- 正文必须暗示"线下可达"（地铁站/地标/营业时间）
- CTA 偏向到店或私信预约

### 2. 服务型（vs 商品型）

- 重点写"过程体验"而不是"产品参数"
- 多用客户故事（脱敏后）
- 价格透明会显著提高转化

### 3. 高客单价

- 信任建设权重 > 卖点展示权重
- 多展示"专业度"细节（资质、过程、思考）
- CTA 偏向私信咨询而非直接成交
